# Consilium Cases

This folder is case memory for Consilium planning, specs, diagnosis packets, implementation plans, handoffs, and retros.

Use it as internal working history. Do not read every case folder as current doctrine, active runtime guidance, or polished onboarding material.

Before acting on a case:

1. Open the case folder's `STATUS.md`.
2. Check the frontmatter `status`, `current_session`, and notes.
3. Treat `plan.md` or session plans as active only when `STATUS.md` says the case is still live.
4. If a folder is missing `STATUS.md` or has uneven older structure, treat it as unclassified working material until verified. Do not assume it is closed, obsolete, or safe to ignore.

Current case conventions live in `../CONVENTIONS.md`. New cases should follow those rules. Old cases are retained for historical context and should not be rewritten just to match today's conventions.
